PFTwitterUtils Class Reference

Inherits from NSObject
Declared in PFTwitterUtils.h

Overview

Provides utility functions for working with Twitter in a Parse application.

This class is currently for iOS only.

Tasks

Interacting With Twitter

  • + twitter

    Gets the instance of the Twitter object that Parse uses.
  • + initializeWithConsumerKey:consumerSecret:

    Initializes the Twitter singleton. You must invoke this in order to use the Twitter functionality in Parse.
  • + isLinkedWithUser:

    Whether the user has their account linked to Twitter.

Logging In & Creating Twitter-Linked Users

  • + logInWithBlock:

    Logs in a user using Twitter. This method delegates to Twitter to authenticate the user, and then automatically logs in (or creates, in the case where it is a new user) a PFUser.
  • + logInWithTarget:selector:

    Logs in a user using Twitter. This method delegates to Twitter to authenticate the user, and then automatically logs in (or creates, in the case where it is a new user) a PFUser. The selector for the callback should look like: (PFUser *)user error:(NSError **)error
  • + logInWithTwitterId:screenName:authToken:authTokenSecret:block:

    Logs in a user using Twitter. Allows you to handle user login to Twitter, then provide authentication data to log in (or create, in the case where it is a new user) the PFUser.
  • + logInWithTwitterId:screenName:authToken:authTokenSecret:target:selector:

    Logs in a user using Twitter. Allows you to handle user login to Twitter, then provide authentication data to log in (or create, in the case where it is a new user) the PFUser. The selector for the callback should look like: (PFUser )user error:(NSError )error
